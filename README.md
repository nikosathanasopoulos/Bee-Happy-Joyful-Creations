# Bee Happy - Χειροποίητα Σαπούνια & Κεριά 🐝🌸

Ένα εκλεπτυσμένο και σύγχρονο ηλεκτρονικό κατάστημα (e-commerce storefront) για χειροποίητα σαπούνια, κεριά και φυσικά προϊόντα περιποίησης, φτιαγμένο με React, Vite και Tailwind CSS v4.

---

## 🛠️ Τεχνολογικό Υπόβαθρο (Tech Stack)

* **React 19** - Βιβλιοθήκη UI για σύγχρονες, γρήγορες και διαδραστικές εφαρμογές.
* **Vite** - Εργαλείο build επόμενης γενιάς με ακαριαίο χρόνο εκκίνησης.
* **Tailwind CSS v4** - Σύγχρονο CSS framework με native postcss plugins για πανέμορφο σχεδιασμό.
* **Motion (framer-motion)** - Για ρευστές, κομψές και επαγγελματικές μεταβάσεις και animations.
* **Lucide React** - Πακέτο εικονιδίων υψηλής ποιότητας.

## 🚀 Τοπική Εκτέλεση (Run Locally)

Ακολουθήστε τα παρακάτω βήματα για να τρέξετε την εφαρμογή στον υπολογιστή σας:

### 1. Προαπαιτούμενα
Βεβαιωθείτε ότι έχετε εγκατεστημένο το [Node.js](https://nodejs.org/) (προτείνεται έκδοση v18 ή νεότερη) και το npm.

### 2. Εγκατάσταση Εξαρτήσεων
Κατεβάστε τον κώδικα και εκτελέστε την παρακάτω εντολή στον φάκελο του έργου για να εγκαταστήσετε όλες τις απαραίτητες βιβλιοθήκες:

```bash
npm install
```

### 3. Εκκίνηση του Development Server
Ξεκινήστε τον τοπικό server ανάπτυξης:

```bash
npm run dev
```

Η εφαρμογή θα είναι διαθέσιμη στη διεύθυνση `http://localhost:3000` (ή στη θύρα που θα εμφανιστεί στο τερματικό σας).

---

## 📦 Οδηγός Build (Production Build)

Για να δημιουργήσετε τη βελτιστοποιημένη έκδοση παραγωγής (production-ready static files), εκτελέστε:

```bash
npm run build
```

Αυτή η εντολή θα δημιουργήσει έναν φάκελο `dist` στην ρίζα του έργου. Ο φάκελος αυτός περιέχει όλα τα στατικά αρχεία (HTML, CSS, JS, εικόνες) έτοιμα για φιλοξενία σε οποιονδήποτε static host (όπως GitHub Pages, Vercel, Netlify, κλπ.).

Για να ελέγξετε το build τοπικά πριν το deploy, μπορείτε να τρέξετε:

```bash
npm run preview
```

## 🐙 Deployment στο GitHub Pages

Η εφαρμογή περιλαμβάνει ένα αυτοματοποιημένο **GitHub Action** για αυτόματο deployment στο GitHub Pages κάθε φορά που κάνετε push στον κλάδο `main`.

### Δομή του Workflow (`.github/workflows/deploy.yml`)
Το GitHub Workflow βρίσκεται στο αρχείο `.github/workflows/deploy.yml` και εκτελεί τα εξής:
1. Ανιχνεύει αλλαγές στον κλάδο `main`.
2. Ρυθμίζει το περιβάλλον Node.js.
3. Εγκαθιστά τις εξαρτήσεις με `npm ci` ή `npm install`.
4. Χτίζει τα στατικά αρχεία με `npm run build`.
5. Ανεβάζει τα αρχεία του φακέλου `dist` στο GitHub Pages χρησιμοποιώντας το επίσημο build & deploy action του GitHub.

### ⚙️ Ρύθμιση στο GitHub
Για να λειτουργήσει σωστά το deployment στο δικό σας αποθετήριο (repository):
1. Μεταβείτε στις **Settings (Ρυθμίσεις)** του repository σας στο GitHub.
2. Στο αριστερό μενού, επιλέξτε **Pages**.
3. Στην ενότητα **Build and deployment**, κάτω από το **Source**, επιλέξτε **GitHub Actions**.
4. Κάντε push τις αλλαγές σας στο branch `main`. Το GitHub Action θα ξεκινήσει αυτόματα και θα δημοσιεύσει τη σελίδα σας!

> 💡 **Σημείωση για το Base Path**: Αν το αποθετήριό σας δεν είναι user page (π.χ. `username.github.io`) αλλά project page (π.χ. `username.github.io/repo-name`), ίσως χρειαστεί να ορίσετε το `base` property στο `vite.config.ts` ώστε να ταιριάζει με το όνομα του repository σας (π.χ. `base: '/repo-name/'`), ή να χρησιμοποιήσετε σχετικές διαδρομές `./` για τις εικόνες και τα assets.
